## Summary of Changes

### Files Modified
1. **`crates/flowstate-runner/src/config.rs`** — Added 3 new CLI flags with validation:
   - `--max-concurrent` (env: `FLOWSTATE_MAX_CONCURRENT`, default: 5) — total parallel runs
   - `--max-builds` (env: `FLOWSTATE_MAX_BUILDS`, default: 1) — concurrent build limit
   - `--shutdown-timeout` (env: `FLOWSTATE_SHUTDOWN_TIMEOUT`, default: 120s) — graceful shutdown grace period
   - Added `validate()` method ensuring `max_builds <= max_concurrent` and both >= 1
   - Added `is_build_action()` helper

2. **`crates/flowstate-runner/src/main.rs`** — Complete rewrite of the poll loop:
   - **Concurrent execution** via `tokio::task::JoinSet` — spawns runs as independent async tasks
   - **Two-semaphore model** — `total_semaphore` (max_concurrent permits) + `build_semaphore` (max_builds permits) ensures builds are serialized while light actions run in parallel
   - **Claim-then-requeue** — when a Build is claimed but no build capacity, re-queues via status update and breaks the claim loop
   - **Graceful shutdown** — SIGINT/SIGTERM sets an `AtomicBool` flag, stops claiming, drains JoinSet with configurable timeout, then force-aborts
   - **Enhanced health endpoint** — returns capacity info (max/active/available) and per-run metadata (run_id, task_id, action, elapsed_seconds) via Axum state
   - **Structured logging** — each spawned task wrapped in `tracing::info_span!` with run_id, task_id, action fields
   - **Salvage under build lock** — timed-out builds run salvage within the same task holding the build semaphore permit

3. **`crates/flowstate-runner/src/lib.rs`** — Added `pub mod run_tracker;`

### Files Created
4. **`crates/flowstate-runner/src/run_tracker.rs`** — Shared state types:
   - `RunTracker` — HashMap-backed tracker with insert/remove/snapshot methods
   - `ActiveRun` — in-progress run metadata (run_id, task_id, action, started_at)
   - `ActiveRunSnapshot` — serializable snapshot for health endpoint
   - `RunResult` / `RunOutcome` — return types from spawned tasks

5. **`docs/CONFIGURATION.md`** — Complete configuration reference documenting all 12 runner variables with examples for sequential, moderate, and high-throughput deployments

### Files NOT Modified (confirmed safe)
